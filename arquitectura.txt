### 💡 Tips Clave para tu Prototipo (Nivel UTN)

1. **Navegación con Enlaces Relativos:** Como es un prototipo, en los menús laterales (Sidebars) de tus HTML, vinculá los archivos usando rutas relativas.
* 
*Ejemplo desde `dashboard.html` de Secretaria:* `<a href="kiosko/3-venta.html">Registrar Venta</a>`.




2. **Un solo CSS para todo:** Usá un único archivo de estilos en la carpeta `/assets/css`. Esto garantiza que todas las pantallas generadas por **Stitch** se vean iguales, respetando la identidad visual de **HexaCode**.


3. 
**Simulación de "Security" (Req 1.i):** Si alguien intenta acceder a una carpeta que no le corresponde (por ejemplo, desde la vista de Alumno al Dashboard de Encargado), podés crear una pantalla simple llamada `403.html` dentro de cada carpeta para demostrar que el sistema "bloquea" accesos no autorizados.


4. 
**Optimización para Internet Inestable:** En la defensa, mencioná que elegiste esta estructura de archivos livianos (Single Page Elements) porque el Wi-Fi de la **UTN-FRRe** suele saturarse y esto permite que el sistema siga siendo funcional bajo carga.
